# medical-codex-chatbot-app
Medical Codex Chatbot App

## FastAPI graph API

A lightweight FastAPI service now wraps the Neo4j graph loaders under `backend/api`. It exposes a consistent interface for the DrugBank, RxNorm, ICD-11, and SNOMED CT datasets that are generated in `backend/generate_graphs/`.

### Run locally

```bash
cd backend/api
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

The API reads the same Neo4j connection variables used by the ingest scripts (`NEO4J_URI`, `NEO4J_USER`, `NEO4J_PASSWORD`, `NEO4J_DB`). To scope requests to a particular ingest run, provide `DEFAULT_DATASET_TAG` in your `.env` (it maps to the `ds` node property) or pass `?ds=...` on individual requests.

All endpoints accept an optional `X-Auth-Token` header. Today it is validated by a stub that always succeeds, but the hook is ready for real authentication logic.

### Key endpoints

- `GET /health` – driver ping checks.
- `GET /datasets` – metadata describing each supported dataset.
- `GET /datasets/{dataset}/nodes/{id}` – fetch node details plus parent/child counts.
- `GET /datasets/{dataset}/nodes/{id}/children?limit=...&offset=...` – page through the current node’s descendants.
- `GET /datasets/{dataset}/nodes/{id}/parents?limit=...&offset=...` – inspect parents/ancestors that reference the node.
- `GET /datasets/{dataset}/search?q=term` – simple substring search across name/title/code + direct ID hits.

Visit `http://localhost:8000/docs` once the server is running to explore the autogenerated OpenAPI documentation.
